# Chat 健身 RPG 记录助手 — 需求文档

## 一、产品定位

本地优先的 Web App 原型，移动端友好的健身训练记录助手。核心体验是用户每次训练后录入训练内容，系统自动分析强化了哪些肌肉群，对应属性加点，像素角色同步成长，达成成就时播放反馈动画。

产品设计遵循反焦虑原则：不做排行榜、不做 PK、不做惩罚式连续打卡，不用负面文案制造压力。

MVP 阶段定位：优先在电脑浏览器运行，前端通过本地 Node API 读写 JSON。界面按移动端响应式设计，真正手机独立使用、PWA 离线安装放到后续阶段。

## 二、MVP 用户故事

用户完成训练 → 手动录入训练数据 → 系统解析动作、匹配肌肉群映射表、计算属性加点 → 像素角色相应部位变强壮 → 达成条件时触发成就动画 → 仪表盘展示体重、训练进度、目标差距。

Chat 对话录入放 P1（先做手动录入 + 预留 adapter 接口）。

## 三、架构约束

1. 数据存储：JSON 文件，通过本地 Node API 读写。Chat、前端、未来飞书机器人，都通过同一套 Node API 写入统一格式的 JSON。

2. Chat 是输入适配器，不耦合核心系统。核心系统接收标准训练记录 JSON，Chat 是其中一个 adapter。未来换飞书/ChatGPT/手动录入，只要输出同一格式即可。

3. MVP 纯本地、单用户。无云同步、无登录、无社交。

## 四、RPG 属性系统

| 类别 | 属性 | 如何加点 |
|------|------|----------|
| 力量 | 胸 | 推类动作（卧推、俯卧撑等） |
| 力量 | 肩 | 推举、侧平举等 |
| 力量 | 背 | 引体向上、划船等 |
| 力量 | 腿 | 深蹲、硬拉等 |
| 力量 | 手臂 | 弯举、臂屈伸等 |
| 力量 | 核心 | 卷腹、平板支撑等 |
| 体能 | 心肺 | 跑步、游泳、骑行等有氧训练 |

每个属性有等级和经验值，做对应训练获得经验，经验满升级。

## 五、肌肉群映射表

系统内置「动作 → 主要肌群 + 次要肌群」映射表，用户可在设置中手动修改。

| 动作 | 主要 | 次要 |
|------|------|------|
| 卧推 | 胸 | 手臂、肩 |
| 深蹲 | 腿 | 核心 |
| 引体向上 | 背 | 手臂 |
| 跑步 | 心肺 | 腿 |
| 游泳 | 心肺 | 背、肩 |

可扩展更多动作。

## 六、训练记录格式（标准 JSON）

力量训练：
```json
{
  "date": "2026-06-05",
  "exercises": [
    {
      "name": "卧推",
      "type": "strength",
      "sets": 3,
      "reps": 10,
      "weight": 60,
      "unit": "kg",
      "note": ""
    }
  ],
  "duration": 45,
  "feeling": "good"
}
```

有氧训练：
```json
{
  "date": "2026-06-05",
  "exercises": [
    {
      "name": "跑步",
      "type": "cardio",
      "duration": 30,
      "distance": 5,
      "distanceUnit": "km",
      "note": ""
    }
  ],
  "feeling": "tired"
}
```

两种 type 可在同一次训练中混合。

## 七、成就系统（三类）

| 类型 | 示例 | 触发条件 |
|------|------|----------|
| 习惯成就 | "每周 3 练" | 本周累计完成 3 次训练 |
| 习惯成就 | "月度铁人" | 本月累计训练 10 次 |
| 习惯成就 | "稳定输出" | 连续 4 周每周至少训练 2 次 |
| 习惯成就 | "欢迎回来" | 停训 2 周以上后恢复训练 |
| 突破成就 | "卧推突破" | 单动作重量突破个人历史 PR |
| 突破成就 | "千吨俱乐部" | 累计总训练重量达 1000 吨 |
| 目标成就 | "减重达人" | 体重达到减重目标值 |
| 目标成就 | "增肌成功" | 体重达到增肌目标值 |

成就基于用户身体数据和训练记录动态计算，第一版预置约 15-20 个。不做严格连续打卡。

## 八、功能模块（底部 4 个 Tab）

**Tab 1：记录**
- 训练录入：动作搜索/选择、力量训练字段（组数、次数、重量）或有氧字段（时长、距离）
- 今日训练总结
- 历史记录列表

**Tab 2：角色**
- 2D 像素风人物展示，随属性成长外观变化
- 属性面板：7 个属性的等级和经验条

**Tab 3：数据**
- 体重趋势折线图
- 训练频率热力图
- 各肌肉群训练量分布
- 目标进度条

**Tab 4：设置**
- 身体数据（身高、体重、目标）
- 成就列表
- 肌肉群映射表编辑
- 数据导出

## 九、像素角色

- 2D 像素风格，正面站立（约 32×48 或 48×64 像素）
- 7 属性对应身体部位可视变化（胸变大、手臂变粗等逐级变化）
- 属性升级时播放短暂动画
- MVP 基础定制：肤色、发型、服装颜色
- MVP 不做装备系统

## 十、成就动画

达成成就时展示全屏像素风庆祝动画，显示成就名称和图标，角色同步更新。

## 十一、技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vite + React + TypeScript |
| 样式 | 移动端优先响应式（Tailwind 或 CSS Modules） |
| 像素角色 | Canvas 2D 或像素 Sprite 图 |
| 数据 API | 本地 Node + Express（极简） |
| 存储 | JSON 文件 |
| 图表 | 轻量库（不引入 ECharts 等重型方案） |
| 动画 | CSS animation 或轻量库 |

## 十二、开发优先级

**P0（第一版必须有）：**
1. 训练手动录入 + 肌肉群映射 + 属性加点计算
2. Node API + JSON 存储
3. 像素角色基础展示 + 属性面板
4. 仪表盘（体重趋势、训练统计、目标进度）

**P1（第二版）：**
5. 成就系统 + 庆祝动画
6. 像素角色成长变化（属性驱动外观）
7. Chat adapter（对话录入训练数据）

**P2（第三版）：**
8. PWA 离线安装 + 手机独立使用
9. 数据导出
10. 飞书 adapter

## 十三、明确不做（MVP）

- 不做后端数据库（MySQL/PostgreSQL）
- 不做用户注册登录
- 不做云同步 / 多设备
- 不做飞书/Telegram 集成（预留接口即可）
- 不做装备系统、技能树
- 不做连续打卡成就
- 不做好友 PK、排行榜
- 不做 Apple Health / 体脂秤对接
